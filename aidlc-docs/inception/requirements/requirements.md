# 要件ドキュメント

## インテント分析サマリー

- **ユーザーリクエスト**: Settingsでお気に入りプロジェクトをチェックボックスで複数選択できるようにし、Add Issueではそのお気に入りの中からプルダウンで選択できるようにする
- **リクエストタイプ**: 機能拡張（Enhancement）
- **スコープ見積もり**: 複数コンポーネント（Settings UI、Add Issue UI、Service Worker、State Manager）
- **複雑度見積もり**: Moderate（既存UIの変更 + 新規ストレージ管理 + コンポーネント間連携）

---

## 機能要件

### FR-1: Settingsパネル - お気に入りプロジェクト選択

**FR-1.1** Settingsパネルに「お気に入りプロジェクト」セクションを追加する。

**FR-1.2** お気に入りプロジェクトセクションでは、Backlog APIから取得したプロジェクト一覧をチェックボックス形式で表示する。

**FR-1.3** ユーザーは複数のプロジェクトにチェックを入れてお気に入りとして登録できる。

**FR-1.4** お気に入りプロジェクトの選択状態はChrome Storage（`chrome.storage.local`）に永続化する。

**FR-1.5** プロジェクト一覧はBacklog APIから動的に取得する（現在のAdd Issueと同様の方式）。

**FR-1.6** プロジェクト一覧の取得はSettingsパネルを開いた時（または「プロジェクトを読み込む」ボタン押下時）に行う。

**FR-1.7** お気に入りプロジェクトの変更は即時または「保存」ボタン押下で保存する。

### FR-2: Add Issueパネル - お気に入りプロジェクトからの選択

**FR-2.1** Add Issueパネルのプロジェクト選択UIを変更し、Settingsで登録したお気に入りプロジェクトのみを表示する。

**FR-2.2** お気に入りプロジェクトが1件以上登録されている場合、プルダウン（セレクトボックスまたは検索可能なドロップダウン）でその中から1つ選択できる。

**FR-2.3** お気に入りプロジェクトが未登録（0件）の場合、エラーメッセージを表示してSettingsへ誘導する。

**FR-2.4** 誘導メッセージには「設定画面へ」ボタンを表示し、クリックするとSettingsパネルに切り替わる。

### FR-3: データ管理

**FR-3.1** お気に入りプロジェクトのデータ構造は以下の通り：
```json
{
  "favoriteProjects": [
    {
      "id": 12345,
      "projectKey": "PROJ",
      "name": "プロジェクト名"
    }
  ]
}
```

**FR-3.2** お気に入りプロジェクトは`chrome.storage.local`の`favoriteProjects`キーに保存する。

**FR-3.3** APIキーが削除された場合、お気に入りプロジェクトのデータも合わせてクリアする。

## 非機能要件

### NFR-1: パフォーマンス
**NFR-1.1** プロジェクト一覧の取得中はローディング表示を行い、UIをブロックしない。

**NFR-1.2** お気に入りプロジェクトの読み込みはサイドパネル起動時に非同期で行う。

### NFR-2: ユーザビリティ
**NFR-2.1** Settingsのお気に入りプロジェクトセクションは既存のAPI設定・テンプレート設定と視覚的に区別できるよう配置する。

**NFR-2.2** チェックボックスリストはプロジェクト名とプロジェクトキーを表示する。

**NFR-2.3** プロジェクト一覧が多い場合はスクロール可能なリストで表示する（最大高さを設定）。

### NFR-3: 信頼性
**NFR-3.1** Backlog APIからのプロジェクト取得に失敗した場合、エラーメッセージを表示してリトライを促す。

**NFR-3.2** ストレージへの保存失敗時は既存のリトライ機構（StateManager）を活用する。

### NFR-4: 保守性
**NFR-4.1** 既存のコードスタイル（キャメルケース、日本語コメント）に従う。

**NFR-4.2** 既存のService Workerのメッセージハンドラーパターンに従い、新しいアクションを追加する。

## ユーザーシナリオ

### シナリオ1: 初回設定
1. ユーザーがSettingsタブを開く
2. 「お気に入りプロジェクト」セクションで「プロジェクトを読み込む」をクリック
3. Backlog APIからプロジェクト一覧が取得される
4. チェックボックスで使用するプロジェクトを複数選択
5. 選択が自動保存（または「保存」ボタン押下）される

### シナリオ2: 課題追加
1. ユーザーがAdd Issueタブを開く
2. お気に入りプロジェクトのプルダウンが表示される
3. プルダウンからプロジェクトを選択
4. 課題種別・件名・説明を入力して課題を作成

### シナリオ3: お気に入り未設定時
1. ユーザーがAdd Issueタブを開く
2. 「お気に入りプロジェクトが設定されていません。設定画面でプロジェクトを選択してください。」というメッセージが表示される
3. 「設定画面へ」ボタンをクリックするとSettingsタブに切り替わる

## 技術的考慮事項

- **既存コードへの影響**: `sidepanel.js`、`sidepanel.html`、`sidepanel.css`、`service-worker.js`を変更
- **新規ストレージキー**: `favoriteProjects`（`chrome.storage.local`）
- **新規Service Workerアクション**: `saveFavoriteProjects`、`getFavoriteProjects`
- **後方互換性**: お気に入りプロジェクト未設定時のフォールバック処理が必要
