# 要件ドキュメント: お気に入りプロジェクト検索 + テンプレート読み込み不具合修正

## インテント分析サマリー

- **ユーザーリクエスト**:
  1. Settings画面の「お気に入りプロジェクト」選択時、プロジェクト名またはプロジェクトキーで検索できる入力欄を追加する（インクリメンタルサーチ）
  2. 不具合修正: お気に入りプロジェクト機能追加後、Settings画面の「課題種別ごとのテンプレート」プルダウンに課題種別が表示されない問題を解消する
- **リクエストタイプ**: 機能拡張 + バグ修正
- **スコープ見積もり**: 単一ファイル中心（`sidepanel.js`, `sidepanel.html`, `sidepanel.css`, i18nメッセージ）
- **複雑度見積もり**: Minimal〜Simple（既存機能の小改修）

---

## 機能要件

### FR-1: お気に入りプロジェクト検索入力欄の追加

**FR-1.1** Settings画面の「お気に入りプロジェクト」セクションのチェックボックス一覧の上部に、検索入力欄を追加する。

**FR-1.2** 検索入力欄は、プロジェクト一覧が読み込まれて表示された時のみ表示する（未読み込み時は非表示）。

**FR-1.3** 検索入力欄に文字を入力すると、入力と同時にチェックボックス一覧をフィルタする（インクリメンタルサーチ）。

**FR-1.4** 検索の一致条件は、プロジェクト名またはプロジェクトキーへの**部分一致**（大文字小文字を区別しない）とする。

**FR-1.5** 検索入力欄をクリアすると、全プロジェクトが再表示される。

**FR-1.6** フィルタ中でも、既に選択（チェック）されているプロジェクトの状態は保持する（チェックをつけた後に絞り込んだり、絞り込みを解除しても、チェック状態は維持される）。

**FR-1.7** 検索結果が0件の場合、「該当するプロジェクトがありません」等のメッセージを表示する。

### FR-2: 「課題種別ごとのテンプレート」プルダウン不具合修正

**FR-2.1** Settingsパネルでお気に入りプロジェクトを保存した直後に、「説明文テンプレート」セクションの「課題種別ごとのテンプレート」プルダウンの選択肢を最新のお気に入りプロジェクトに基づいて再構築する。

**FR-2.2** お気に入りプロジェクトが0件になった場合、「課題種別ごとのテンプレート」プルダウンはデフォルト選択肢（「デフォルト（全種別共通）」）のみの状態にリセットする。

**FR-2.3** お気に入りプロジェクトが追加された場合、新たに追加されたプロジェクトの課題種別をプルダウンに追加する。

**FR-2.4** 既存の `loadIssueTypesForTemplateEditor()` を再利用し、重複実装を避ける。

## 非機能要件

### NFR-1: パフォーマンス
**NFR-1.1** インクリメンタルサーチのフィルタ処理は、クライアントサイド（JavaScript）で行い、Backlog APIへの追加リクエストは発生させない。

### NFR-2: ユーザビリティ
**NFR-2.1** 検索入力欄は、`placeholder` 属性で検索方法を明示する（例: 「プロジェクト名またはキーで検索」）。

**NFR-2.2** 既存のデザイン（border、padding、フォーカス時の緑色ハイライト）と統一する。

### NFR-3: 国際化
**NFR-3.1** 新規UI文言は `_locales/ja/messages.json` と `_locales/en/messages.json` の双方に追加する。

### NFR-4: 保守性
**NFR-4.1** 既存のコードスタイル（キャメルケース、日本語コメント）に従う。

## ユーザーシナリオ

### シナリオ1: 検索でプロジェクトを絞り込む
1. ユーザーがSettingsタブの「お気に入りプロジェクト」セクションで「プロジェクトを読み込む」をクリック
2. プロジェクト一覧がチェックボックス形式で表示される
3. 検索入力欄に「backend」と入力
4. 「backend」をプロジェクト名またはプロジェクトキーに含むプロジェクトのみが一覧に表示される
5. 絞り込まれた一覧から目的のプロジェクトのチェックボックスを選択
6. 検索欄をクリア → 全プロジェクトが再表示され、先ほどチェックしたプロジェクトはチェック状態を維持
7. 「保存」をクリック

### シナリオ2: テンプレートプルダウン不具合修正後の動作
1. ユーザーがSettingsタブでお気に入りプロジェクトを選択・保存
2. 「説明文テンプレート」セクションを下にスクロール
3. 「課題種別ごとのテンプレート」プルダウンを開く
4. お気に入りプロジェクトから取得した課題種別が一覧表示される
5. 任意の課題種別を選択するとエディタにテンプレートが表示される

## 技術的考慮事項

### 不具合原因の分析
- 現在、`loadIssueTypesForTemplateEditor()` は `init()` 内の `initializeTemplateEditor()` でのみ呼び出されている。
- サイドパネル初期化時、`loadFavoriteProjectsFromStorage()` は `init()` の最後に呼ばれるが、この時点で `initializeTemplateEditor()` は既に実行済み。
- そのため、初回起動時にはお気に入りデータがまだロードされていない状態で `loadIssueTypesForTemplateEditor()` が実行され、課題種別プルダウンが空のままになる。
- また、お気に入りプロジェクトを保存した後も `loadIssueTypesForTemplateEditor()` が再実行されないため、プルダウンが更新されない。

### 修正方針
1. `init()` の初期化順序を見直し、`loadFavoriteProjectsFromStorage()` を `initializeTemplateEditor()` より**前に**呼び出す
2. `saveFavoriteProjects()` 成功時に `loadIssueTypesForTemplateEditor()` を再実行する
3. 検索入力欄は既存の `renderFavoriteProjectsList()` を変更してフィルタに対応させる

### 変更対象ファイル
- `sidepanel/sidepanel.html` (検索入力欄のマークアップ追加)
- `sidepanel/sidepanel.css` (検索入力欄のスタイル)
- `sidepanel/sidepanel.js` (フィルタロジック、初期化順序、保存後の再描画)
- `_locales/ja/messages.json`, `_locales/en/messages.json` (新規文言)

### 後方互換性
- 既存のお気に入りプロジェクト保存データ（`chrome.storage.local.favoriteProjects`）には影響なし
